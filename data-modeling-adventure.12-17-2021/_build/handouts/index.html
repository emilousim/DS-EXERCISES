Data Modeling Adventure

Introduction

In this exercise you will get additional practice with defining an appropriate model for an existing set of data.

For this section, include your answers in a new file called discussion.txt.

  1. Define normalization in your own words. Be sure to include why normalization is important.

  2. What is the difference between 1NF, 2NF, and 3NF?

Be sure to add, commit and push your answers to Github.

Part 2: Food Environment Atlas

For this section, you will need to log in to data.world and download the USDA Food Environment Atlas dataset.

One skill of data modeling is taking an existing dataset that is organized in files and creating a relational model that can be used in a SQL database. In this section, you will need to carefully review the data in the existing dataset for USDA’s Food Environment. While the data is organized within files, the existing structure would not be the perfect data model if you were to seed this data directly into a SQL database.

After you have reviewed the data in detail, start making a list of tables for your database. As you consider whenther something should be a table, keep in mind that it’s helpful for tables to represent objects or entities, rather than ideas or concepts.

After you have a list of tables, make a column list for each table. You should try to include all data from the dataset into your data model. In your colummn list, also include the datatype for each column.

Lastly, consider the relationships between tables. Do you have a one-to-many relationship in your data model? Is there a need for any many-to-many relationships? Notate the relationships using lines between your tables.

Once you have finished your draft, go to DB Designer and create a final version of your data model.

When you are finished, export your finished database as a PDF and add it to your repository for this lab exercise. Remember to push to Github!
